## t041 ‚Äî SKU lookup from a fuzzy product description ("sku only")
- **Instruction (run1):** "sku for 2x3ah makita dhs680 saw kit pls. sku only." (each run a different product)
- **Type:** lookup
- **Runs analyzed:** 9
- **What the agent did:** Bootstrap ‚Üí SQL `product_variants` (almost always returns EMPTY ‚Äî either MS-SQL "cluster down" timeout or silent empty) ‚Üí falls back to `harness.search`/`find`/`list` on `/proc/catalog`, finds the variant family, reads the record(s), returns the SKU. Runs diverged on whether the description resolved to exactly one variant (answer the SKU) or several (the agent chose `OUTCOME_NONE_CLARIFICATION` and asked which).
- **Answers across runs:** unambiguous‚Üísingle SKU: PT-SAW-MAK-DHS680-3AH, PT-MOW-EIN-GECM36-2X4, PT-CMP-AIR-CA240-6, PT-GRD-MET-W18-125-FLAT, PT-SND-BOS-GEX125-CASE (5 runs). - **Outcomes across runs:** OK √ó5, NONE_CLARIFICATION √ó4.
- **Variance:** FLIPS (between answering and refusing-to-disambiguate; driven by instruction, but the clarification choice is a judgment risk).
- **Inferred verdict:** MIXED ‚Äî the 5 single-SKU runs are well-grounded and almost certainly correct (each read the matching record and the attributes line up: tank_l=24/6, flat-head body, case-and-25-disc kit). E.g. "kit with charger" ‚Äî BOTH 3AH and 5AH say "...batteries and charger", so a clarification is defensible, but "not the 4Ah" Einhell with BODY+2X5 remaining is genuinely two-way. - **Failure codes:** F0 (5 runs), F12 over-clarification / under-commitment (4 runs), F3 minor overcite (1 run).
- **Steering opportunity:** Add a rule: "For a SKU lookup from a fuzzy phrase, pick the single best-matching variant and answer its SKU; only emit NONE_CLARIFICATION when ‚â•2 variants match ALL stated attributes equally. A qualifier like 'kit with charger' or 'not the 4Ah' is a *filter*, not a request for a menu ‚Äî apply it and return the unique survivor."
